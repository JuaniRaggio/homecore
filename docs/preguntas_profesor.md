# Preguntas para el profesor - Features opcionales (grupos de 3)

## Preguntas opcionales - Enunciado

- **18-19**: Gestionar y consultar hogares
- **20**: Vincular habitaciones a hogares
- **21**: Enviar notificaciones (sitio web opcional)
- **22**: Restringir acceso a dispositivos/rutinas/habitaciones/hogares (opcional ambas plataformas)
- **23**: Consultar consumo electrico (opcional ambas plataformas)
- **24**: Planificar ejecucion de rutinas (opcional ambas plataformas)

Ademas, en dispositivos (punto 9):
- Aspiradora y parlante: minimo 1 grupo de 3
- Heladera, horno y aire acondicionado: minimo 2 grupos de 3

## Preguntas

### 1. Valor de las features opcionales en la evaluacion
Las funcionalidades opcionales suman puntos extra en la nota, o simplemente no restan? Es decir, un grupo de 3 que implemente todas las opcionales puede aspirar a una mejor nota que uno que solo implemente las obligatorias?

### 2. Prioridad entre opcionales
Si tuvieramos que elegir un subconjunto de opcionales por limitaciones de tiempo, cuales consideran mas valiosas o relevantes para la evaluacion? Hay alguna que se valore mas que otra?

### 3. Profundidad vs amplitud
Es preferible implementar menos features opcionales pero con mayor profundidad y mejor justificacion de diseno (desde HCI), o conviene cubrir la mayor cantidad posible aunque sea con menor nivel de detalle?

### 4. Dispositivos adicionales
Respecto al punto 9, los dispositivos del segundo grupo (heladera, horno, aire acondicionado) tienen controles mas complejos. Implementar alguno de estos suma significativamente respecto a quedarse solo con los del primer grupo?

### 5. Gestion de hogares - complejidad vs beneficio
La gestion de hogares (puntos 18-20) agrega una capa jerarquica completa (hogar > habitacion > dispositivo). Vale la pena implementarla considerando que agrega complejidad a la navegacion y puede afectar la usabilidad, o prefieren que nos enfoquemos en pulir las funcionalidades obligatorias?

### 6. Criterio de evaluacion para la primera entrega
En la primera entrega, se evalua que las features opcionales esten disenadas en los prototipos, o solo se evaluan las obligatorias? Queremos saber si debemos incluir las opcionales en los wireframes/mockups desde ahora.
